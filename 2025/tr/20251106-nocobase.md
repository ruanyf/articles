
## nocobase

## AI 在普通业务场景里如何落地


周刊在 2022 年就分享过的一个开源无代码产品 [NocoBase](https://www.nocobase.com/cn/)，刚刚发布了 AI 员工的能力, 简单来说就是让 AI 无缝集成到业务系统中，成为团队和业务的一部分，从而让 AI 在普通业务场景中协助人类工作。

![AI-Employees](https://static-docs.nocobase.com/20251105-1446562.jpeg)

比如在一个制造业企业中，AI 员工 Vera 可以自动去调查一个客户的背景资料，员工 Viz 可以自动分析各种维度的数据，员工 Jade 可以根据采购需求去各供应商平台获取数百个备选商品并给出最佳采购策略。

这些问题都很小，不像编程那样是巨大的产业，但在普通企业中浪费大量人力。而 NocoBase 也并没有做颠覆性的创新，只是在大模型和无代码平台的基础上做了很好的融合。

可以在它的 [官网](https://www.nocobase.com/cn/) 上开通 demo 试一下，也可以从  [GitHub](https://github.com/nocobase/nocobase) 获取源代码自己部署。

NocoBase 是一个高度可扩展的无代码/低代码开发平台，使用户能够快速构建各种业务应用程序。它以数据模型驱动的核心为基础，使用户能够通过简单的配置来设计和管理复杂的工作流程。

NocoBase 注重灵活性和可扩展性，为需要在定制开发和无代码应用之间取得平衡的团队提供强大的解决方案。用户还可以通过插件扩展其功能，以满足更高级的开发需求。

NocoBase 是一款数据模型驱动的工具，允许用户根据业务需求定义复杂的数据结构。它不仅支持关系型数据，还支持更多种类的数据类型，能够应对复杂的业务场景。NocoBase 灵活的数据权限控制，结合前端模块，可以显著提高效率。过去需要一整天开发才能完成的任务，现在只需几分钟即可完成，并带来令人瞩目的成果。

目标：帮助开发者（甚至非技术用户）快速构建应用程序，从而减少对传统软件开发的依赖。

拥有强大的数据管理能力和灵活的插件架构。其插件架构允许用户根据需要扩展功能，使其适用于复杂的业务流程管理、数据访问控制以及高度定制化的应用场景。

NocoBase 是一个数据模型驱动的平台，具有强大的数据管理功能，包括：

- 数据模型驱动设计：NocoBase 采用数据模型驱动的方法，用户可以通过配置数据模型来定义应用程序的基本结构。数据源管理是一个独立的插件，用于管理不同的数据源和表。支持的数据源包括 NocoBase 的主数据库（支持 MySQL、PostgreSQL 和 SQLite）、外部 MySQL、MariaDB 和 PostgreSQL 数据库。
- 复杂数据关系：NocoBase 支持对复杂数据关系（例如一对一、一对多和多对多）进行建模和管理。开发人员可以建立表之间的关系，并通过界面或 API 轻松查询或操作相关数据。
- 精细化访问控制：NocoBase 提供精细化的权限控制，支持字段级和行级的数据访问设置。开发人员可以为不同的角色和用户组配置访问权限，从而确保数据安全性和合规性。

NocoBase 采用所见即所得的界面配置模式。点击右上角的界面配置按钮（橙色区域）即可在配置模式和使用模式之间切换。

NocoBase 提供多种方式实现与第三方系统的集成：

第三方数据库作为数据源：NocoBase 支持集成外部数据库（例如 MySQL、PostgreSQL）作为数据源，使开发人员能够直接在平台内访问和操作外部数据。
第三方 API 集成：开发人员可以将其他系统的 API 集成为数据源，配置接口以获取或推送数据，从而促进与现有系统的无缝数据交互。
单点登录 (SSO)：NocoBase 支持 SSO，可与第三方身份验证系统集成，使用户能够通过统一的身份验证系统访问多个应用程序。
嵌入式集成：NocoBase 支持在其界面中嵌入第三方系统，从而实现系统间更紧密的集成。
多应用集成：不同的 NocoBase 应用可以相互交互，实现跨应用的数据交换和协作，适合构建分布式应用系统。

NocoBase 为复杂的安全需求提供细粒度的多级访问控制：

多级权限管理：支持对表、字段和记录进行权限控制。开发人员可以为不同的角色和用户组配置不同的数据访问权限，从而实现精准的数据管理。例如，某些用户组可能只能查看特定的字段或记录，以防止未经授权的访问。

角色和用户组管理：允许开发人员创建具有特定权限的自定义角色和用户组，使配置灵活且能够适应不断变化的业务需求。

数据访问审计和日志记录：NocoBase 还会记录用户操作，以便进行安全审计和合规性检查。
