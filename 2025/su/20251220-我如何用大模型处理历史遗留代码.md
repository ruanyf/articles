# 我如何用大模型处理历史遗留代码，暨 MiniMax M2.1 试用感受

我不太用大模型生成全新的项目，因为总感觉，生成出来的是它的项目，不是我的项目。

历史遗留代码，没有文档，也没有作者的说明，所有一切都要靠自己琢磨。而且，它使用的技术栈和工具都过时了，你还要去把它们都翻出来，重温这些过时的技术栈。

更可怕的是，有些情况下只有构建产物，没有源代码。这叫人怎么改？以前，这种情况是没法处理的。

但是，现在不一样了，扔给大模型就可以了，它甚至对构建工程进行逆向工程，最适合处理逆向工程。

---

【内容简介】

本文展示作者日常如何使用大模型，处理历史遗留代码，一方面作为大模型的使用教程，另一方面展示 MiniMax M2.1 分析代码、修改代码、添加功能的能力。

【文章结构】

一、引言

作者收到读者留言，希望介绍日常如何使用大模型。由此引出个人的使用习惯：大模型多用于处理历史遗留项目，而不是生成全新项目。

本文介绍我的日常用法，希望大家可以补充你是怎么用的。

二、模型和工具的选择

AI 编程工具，我目前主要使用 Claude Code，原因是命令行用起来更方便、直观，还可以跟其他传统工具相结合。

日常的主要编程模型，我用的是 MiniMax M2，因为它功能强、价格便宜，有29元的包月，适合大量使用，反复试错。

如果自费使用 Claude 模型，20美元的Pro套餐不够用，200美元的Max套餐觉得贵。

三、MiniMax M2.1 模型

MiniMax 本周刚升级了 M2 模型，新的 M2.1 模型据说功能更强。（此处出插入介绍）。

我就用这个新模型，展示我的日常用法，顺便测一下这个模型。

需要声明的是，大模型发展到今天，简单的任务已经越来越难测出它们的强弱了。可以这样认为，对于一般的编程任务，目前主流的编程模型都是够用的，没必要只把目光盯在国外大厂最新的旗舰模型上面。

四、编程任务介绍

本次选择的是我正在用的一个老仓库，微信公众号的可视化编辑器 wechat format，它能把 Markdown 代码转成微信公号的格式。

原始作者六年前就停止更新，没有文档，使用的是旧版本的 Vue.js，而且代码是经过编译工具处理过的，不熟悉这个框架的开发者接手这个仓库很费劲。

五、代码解释

使用 MiniMax M2.1，让它生成这个仓库的概述，并且解释 index.html 的代码。

六、代码修改

使用 MiniMax 2.1，修改某个网页样式，把链接从红色改成蓝色。

七、增加功能

使用 MiniMax 2.1，为这个编辑器增加一个功能，将微信公众号的链接也加入文尾的链接列表。

八、总结

以前要做上面这些任务，你必须通读老项目的代码，彻底搞懂后才能动手修改。

但是用了大模型，根本不需要读代码了，直接扔给模型，只要描述你的需求就可以了，它五分钟就能解决，很适合处理历史遗留项目。

上面的任务，我的提示词很简单，只有一句话，MiniMax M2.1 都是一次性就做对了，而且带有很丰富的解释。我对它比较满意，认为大家日常工作也可以考虑使用它。
