# 大模型比拼：MiniMax M2 vs GLM 4.6 vs Claude Sonnet 4.5

一、

上个月，我写了[一篇文章](https://www.ruanyifeng.com/blog/2025/10/glm-4.6.html)，比较了两个大模型。

就有人留言，两个模型太少了，能不能加入其他模型？

正好上个礼拜（10月27日），MiniMax 公司[发布了](https://www.minimaxi.com/news/minimax-m2) M2 模型，代表了国产大模型的最新水平。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110301.webp)

我就想，可以测测它的实战效果，跟智谱公司的 GLM 4.6 和 Anthropic 公司的 Claude Sonnet 4.5 对比一下。

毕竟它们都属于目前最先进的编程大模型，跟我们开发者切身相关。

二、

先要说明，其实我不太熟悉 [MiniMax 公司](https://www.minimaxi.com/)，它比较低调。

我只知道，这家公司专门研发大模型，有文本模型、视频模型、音频模型等等，但都不是特别热门。我就没有特别关注。

上个礼拜，我在滑推特的时候，看到一些老外在议论（[1](https://x.com/reach_vb/status/1982705125157126590)、[2](https://x.com/ivanfioravanti/status/1982103760819064967)、[3](https://x.com/simonw/status/1982825075553493136)），这才知道 MiniMax 发布了新的旗舰模型 M2。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110302.webp)

上面[这个人](https://x.com/reach_vb/status/1982705125157126590)是 HuggingFace 大模型社区的负责人，提到了 M2 模型在 Artificial Analysis 性能评比之中排名第五，开源模型第一。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110303.webp)

当天的 HuggingFace 热度榜上，它也是第一名。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110304.webp)

我就来了兴趣，准备好好试用一下。

三、

根据 MiniMax 公司的说明，**M2 模型的编程能力特别强**，是目前最优秀的编程模型之一。

大家知道，国际上最流行的编程模型现在是 Claude Sonnet 4.5，国内的 GLM 4.6 模型也很强，我就把它们三个放在一起对比。

简单起见，我就直接在官方的网页版（[国内版](https://agent.minimaxi.com/)，[国外版](https://agent.minimax.io/)）上运行测试，大家可以跟着一起动手试试看。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110305.webp)

网页版是官方的智能体产品 [MiniMax Agent](https://www.minimaxi.com/news/minimax-agent)，底层用的就是 M2 模型。

现在是**免费期**，API 调用不需要花钱。不过，它的定价不算高，百万 tokens 的输入/输出价格是 2.1元/8.4元人民币，官方宣传只有 Claude 价格的8%。

对于需要的同学，我也列一下，它的文档仓库在 [GitHub](https://github.com/MiniMax-AI/MiniMax-M2)，官方文档有 [API 调用指南](https://platform.minimax.io/docs/guides/text-generation)（兼容 OpenAI 和 Anthopic 格式），[模型下载](https://huggingface.co/MiniMaxAI/MiniMax-M2)在 HuggingFace，有条件可以本地部署使用。

四、

我使用的测试题是，著名程序员西蒙·威利森（Simon Willison）测试 Cluase Sonnet 4.5 的题目，他的网站有[测试结果](https://simonwillison.net/2025/Sep/29/claude-sonnet-4-5/)。

我之前用这些题目，测过智谱公司的 GLM 4.6 模型，大家可以[参考](https://www.ruanyifeng.com/blog/2025/10/glm-4.6.html)。

下面就是 MiniMax M2 的表现。

五、

第一题，测试模型的代码理解和运行能力。

我要求 M2 模型读取一个仓库，运行里面的代码，并返回结果，提示词如下。

> 拉取代码仓库 https://github.com/simonw/llm ，然后通过下面的命令运行测试用例。
> 
> pip install -e '.[test]'  
> pytest

上面的提示词就是让模型运行一个 Python 仓库里面的测试用例。

Minimax Agent 显然内置了沙盒，可以在隔离环境下运行代码（下图）。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110306.webp)

整个运行过程大约三分钟，然后它给出了正确结果：运行通过了466个测试用例。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110307.webp)

令我惊喜的是，除了运行结果，它还给出了一段覆盖率分析（下图），指出这些用例覆盖了哪些方面。我没在其他模型见过这个分析。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110308.webp)

完整的对话看[这里](https://agent.minimaxi.com/share/329839918691041?chat_type=1)。

六、

第二题，测试大家最关心的代码生成能力，看看它能不能按照要求生成应用程序。

我还是使用上面的仓库，要求 M2 为其增加一个功能，不仅需要增加代码，还需要修改数据库结构，并增加配套的测试用例。

> 1、代码仓库 https://github.com/simonw/llm 是一个 AI 对话应用，它将用户的提示和 AI 的响应存储在 SQLite 数据库中。
>
> 2、它目前使用线性集合，保存单个对话和响应。你尝试在响应表中添加一个 parentresponseid 列，并通过该列将对话的响应建模为树状结构。
> 
> 3、编写新的 pytest 测试用例，验证你的设计。
> 
> 4、编写一个 tree_notes.md 文件，首先将你的设计写入该文件，然后在运行过程中将该文件用作笔记。

这是比较复杂的任务，运行的时间稍微长一点。

它第一次读取 GitHub 仓库时，没有成功，出现了我意外不到的一幕。它自动切换到第三方的 deepwiki.com 去获取仓库。后面，分析数据库结构时，它又切换到 datasette.io 去分析 SQLite 数据库。

这种第三方云服务的自动切换，我也是第一次见，可惜没来得及截图。

任务完成后，它给出了一段总结（下图），详细描述了它做了哪些事，包括数据库修改、新增的测试用例等等。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110309.webp)

它甚至增加了一个示例文件，演示新增的功能怎么用，还有一个示例图，演示修改后的对话结构（下图），我的提示里面可没要求它这么做。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110310.webp)

完整的对话看[这里](https://agent.minimax.io/share/329995155439860?chat_type=1)。另外，它的[官网画廊](https://agent.minimaxi.com/)还有很多它生成的应用，可以看看。

七、

第三题，测试它的理解和推理能力，就是西蒙·威利森发明的著名“鹈鹕骑自行车”场景。

> 生成鹈鹕骑自行车的 SVG 图片。（Generate an SVG of a pelican riding a bicycle）

这是现实中不存在的情景，全靠模型自己推理出来。理解能力越强，生成的图像就越像真的。

下面就是它生成的结果，完整的对话看[这里](https://agent.minimaxi.com/share/329832235541493?chat_type=1)。

八、

测试就到这里，一共三道题。本文主要关注 MiniMax M2，至于 [GLM 4.6](https://www.ruanyifeng.com/blog/2025/10/glm-4.6.html) 和 [Claude Sonnet 4.5](https://simonwillison.net/2025/Sep/29/claude-sonnet-4-5/) 的结果，大家可以看它们各自的链接，进行比较。

我必须诚实地说，MiniMax M2 的表现超出了我的预期。

最打动我的地方，还不是它的处理结果本身，而是它处理问题的方式，明显对用户很友好，会显示一些帮助你理解的辅助步骤，让你觉得很易用（accessible）也很易懂。

我倾向于相信各种评测结果，确实是它的真实实力。再考虑到它的 API 现在是免费使用，我会在平时工作中使用它，也推荐大家试试看。
