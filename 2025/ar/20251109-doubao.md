我使用的工具是 Claude Code，它是眼下最流行、公认功能最强的 AI 终端客户端。

二、

我要分享的方法，完全不影响 Claude Code 本身，它还能正常使用。

也就是说，执行 `claude` 命令，调用的还是原来的 Claude Sonnet 4.5 模型。

```bash
$ claude
```

我会新增一个 `claude-doubao` 命令，执行后底层模型就切换了。

```bash
$ claude-doubao
```

这时，Claude Code 的功能一样都不少，但是使用的模型已经变成了 Doubao-Seed-Code。

上面两个命令，可以同时使用，互不影响。你还能继续添加其他模型，每一个模型都有自己的调用命令，最大程度方便测试新模型。

三、

开始之前，我先简单介绍一下 Doubao-Seed-Code，优秀的国产模型值得好好宣传。

这个新模型属于豆包大模型系列，有着该系列迄今为止最强的编程表现。

它的基础模型是 Doubao Seed 1.6 thinking，专门加强了编程训练，内部型号是基础模型的 code preview 分支。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025111001.webp)

根据 [SWE-bench 的评测](https://www.swebench.com/)，它和同是字节旗下的智能 IDE 产品 Trae 的组合，在 SWE-Bench-Verified 的排行榜位列第一。

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025110907.webp)

我看中这个模型，有三个原因。

（1）它是大厂主推的产品，模型质量、性能表现、响应速度都有保证。

（2）它原生兼容 Anthropic API，可以丝滑接入 Claude Code，不用任何调整，是很好的平替。

（3）它价格便宜，API 调用有包月价，这可不是每个模型都有的。一般强度使用，第一个月9.9元（续费40元/月），官方的宣传是“属于国内最低价格”。

大家可以在[火山方舟的模型广场](https://console.volcengine.com/ark/region:ark+cn-beijing/model/detail?Id=doubao-seed-1-6-thinking)，找到这个模型，开通 API Key，后面要用到。

四、

下面就来演示，Doubao-Seed-Code 如何快捷接入 Claude Code。

如果你已经安装了 Claude Code，下面的方法是完全不影响的，还是正常使用。

如果还没有安装 Claude Code，但想体验它的 Claude Sonnet 4.5 模型，可以根据[官方文档](https://code.claude.com/docs/en/setup)进行安装，跟下面的方法也不冲突。

第一步，新建一个项目目录`claude-model`，在里面安装一个单独的 Claude Code。

```bash
$ mkdir ~/claude-model
$ cd ~/claude-model
$ npm init -y
$ npm install @anthropic-ai/claude-code
```

然后，新建一个子目录 `.claude-doubao`，用来存放豆包的配置文件和缓存。

```bash
$ mkdir .claude-doubao
```

第二步，新建一个子目录 `bin`，用来存放可执行脚本。

```bash
$ mkdir ~/claude-model/bin
```

然后，要把这个 `bin` 目录放入 `PATH` 变量，让系统可以找到里面的命令。

`PATH` 变量的设置，macOS 可以修改 `～/.bash_profile` 文件，Linux 可以修改 `~/.bashrc` 文件。

改动 `PATH` 变量的值，大概是下面这样。

```bash
export PATH="$HOME/claude-model/bin:$PATH"
```

完成后，别忘了重启终端。

第三步，在上一步创建的 `bin` 目录里面，新建一个名为`claude-doubao` 的脚本，用来调用豆包模型。

```bash
$ touch ~/claude-model/bin/claude-doubao
```

在这个 `claude-doubao` 脚本里面，输入下面的内容。

```bash
#!/usr/bin/env bash
# Wrapper for Claude Code CLI using Doubao API

CLAUDE_BIN="$HOME/claude-model/node_modules/.bin/claude"

# Inject API credentials
export ANTHROPIC_AUTH_TOKEN="YOUR_DOUBAO_API_KEY"
export ANTHROPIC_BASE_URL="https://ark.cn-beijing.volces.com/api/compatible"
export ANTHROPIC_MODEL="doubao-seed-1-6-thinking-code-preview"
export API_TIMEOUT_MS=3000000

# Keep a separate config dir (optional)
export CLAUDE_CONFIG_DIR="$HOME/claude-model/.claude-doubao"

exec "$CLAUDE_BIN" "$@"
```

然后，将它变成可执行脚本。

```bash
$ chmod +x ~/claude-model/bin/claude-doubao
```

这样就完成所有设置了，只需要这三步。

现在测一下。

```bash
$ claude-doubao --version
```

![](https://cdn.beekka.com/blogimg/asset/202511/bg2025111003.webp)

以后，如果调用 Doubao-Seed-Code 模型，就用 `claude-doubao` 命令；如果调用原始的 Claude Code，就用 `claude`命令。

```bash
# 调用 Doubao-Seed-Code
$ claude-doubao

# 调用原始模型
$ claude
```

这种方法是通用的，其他模型也可以这样添加。
