## 第二波“开放 API”浪潮

https://brandur.org/second-wave-api-first

十五年前的2011年，云服务开始兴起，诞生了第一波“开放 API”的浪潮。

> Facebook：发布了 Open Graph API，为深入探索它的平台数据提供了极其强大的功能。
> 
> Twitter：API 几乎完全开放。甚至不需要 OAuth 令牌——只需用户名/密码就能进行身份验证，访问几乎所有内容。
> 
> GitHub：非常出色的 API 设计，开发者可以访问任何需要的功能，并且尝试了 API/HATEOAS 等具有前瞻性的想法。

当时的想法是，开放的 API 有助于用户和第三方，深入使用平台，添加插件和扩展程序，从而推动增长，提高产品留存率和用户满意度。

那时还出现了一个叫做 ProgrammableWeb 的网站（现已关闭），它的名字就代表了当时人们的信念，互联网应该是可以编程的，把各种数据拼接在一起。

但是，后来的发展跟人们的预想完全相反。

平台发现，开放 API 不仅无法盈利（没法做广告），反而会带来用户流失，自家的数据帮助了其他公司的生意。于是，它们纷纷改变做法，限制和关闭 API，开始将用户保留在自家的“围墙花园”里面。

> **FaceBook**：自从剑桥分析公司事件（窃取用户数据，用于广告投放）后，Graph API 的功能受到了极大的限制，接入需要通过严格的应用审核流程。
>
> Twitter：新功能不再提供 API，数据流的访问权限也逐渐被关闭。第三方客户端受到限制，最终被彻底禁用。
>
> Instagram：完全弃用了之前公开的 API，要么通过官方应用使用 Instagram，要么就完全不用。
>
> GitHub：虽然保持开放的 API，但也采取了一定程度的管控。使用必须经过身份验证，并且实施了严格的速率限制，通过 API 来构建完整的第三方应用程序变得很困难。

2025年的年中，事情开始出现变化，功能强大的大模型迅速崛起，成为程序员最有用的新工具，主导了所有的媒体讨论。

但是，如果大模型不能执行代码，用处就不大。它的最大价值来自自动化（比如龙虾），而不是内容生成，因为只有通过自动化，大模型才能大量用于企业和生产环境。

自动化需要大模型能够调用其他工具，这意味着，那些工具必须开放自己的 API。

突然之间，API 不再是负担，而成为重要的接入渠道，让各种 Agent 能够代表用户完成工作。尤其对于产品差异较小的市场，API 的可用性在不久的将来很可能成为决定产品胜败的关键。

同样两家银行，一家有 API，另一家没有。后者就很可能被市场淘汰，因为大模型无法接入它，也就无法自动化转账。

种种迹象表明，第二波“开放 API”浪潮正在到来。

各种平台开始意识到，谁尽快向 AI 开放 API，谁就将抢占先机。所以，腾讯才会在龙虾 OpenClaw 爆红以后，以最快的速度开放了微信的接口，让龙虾可以向微信发消息。

其他产品也类似，早先就纷纷通过 MCP 和 Skill，开放自家的操作接口。

新一波的开放浪潮，将远比上一波更声势浩大、更易用。

（1）这次开放的不仅是云服务，还有很多人们日常需要的生活服务：外卖、电商、银行……甚至还有很多意想不到的服务，比如大模型预订餐厅和球馆。

（2）不再需要编程，人们将使用自然语言输入，由大模型翻译后调用 API。

（3）API 的使用对象，以前是应用程序，现在是消费者本人。

（4）API 的使用目的，以前主要是获取数据，现在是代表用户行事。

（5）结论：随着龙虾类 Agent 产品和运作模式的日益普及，API 的可用性有望成为竞争优势，帮助提供商在竞争中脱颖而出。结果很可能是，开放 API 在全球范围内的爆炸式增长。

## 前端是不是重复劳动？

我看到一个开发者[说](https://jonno.nz/posts/what-if-your-browser-built-the-ui-for-you/)，前端本质上是相同的工作：向用户展示一些数据，并让用户处理这些数据。

他觉得，没必要重复解决同样的问题。

他就做了一个“[自适应浏览器](https://github.com/jonnonz1/adaptive-browser)”。它通过 AI 自动生成前端 UI，后端只需要提供数据，以及网页用途的描述。

不知道这会不会是前端的归宿？

Adobe 公司的主要产品是“创意云”套件（Creative Cloud），里面包括许多著名的软件，比如 Photoshop、Illustrator、Premiere。

![](https://cdn.beekka.com/blogimg/asset/202604/bg2026040404.webp)

![](https://cdn.beekka.com/blogimg/asset/202604/bg2026040403.webp)

一个应用程序为什么要修改系统文件呢？

这么著名的软件，居然干这种事情，让人无语。

1、[行为经济学诱饵](https://www.sina.cn/news/detail/5279286413232198.html)

行为经济学家丹·艾瑞里，有一天闲逛《经济学人》官网。

他在订阅页面上，看到了三个选项：

> A. 电子版——59美元。
> B. 纸质版——125美元。
> C. 纸质版＋电子版——125美元。

他愣住了。

B 和 C，价格一模一样。一个只给纸质版，一个纸质版加电子版全送。谁会选 B？

傻子都不会啊。但艾瑞里没有笑，马上意识到这是一个绝妙的设计。

他拿着这三个选项，走进了 MIT 的课堂，做了一个实验，让100个学生面对这三个选项进行选择。

结果：16%选了 A。 0%选了B。 84%选了C。总收入：11,444美元。

跟预想的一样，没有一个人选B。

然后艾瑞里做了一件小事：他把 B 删了，只留 A 和 C。

逻辑上，一个从来没人选的东西，删掉它不应该影响任何结果，对吧？

结果出来了：68%选了 A，32%选了 C。总收入暴跌到8,012美元。

一个从来没人选的选项 B，自己一份都没卖出去，却在暗中帮旁边的最贵的 C 套餐，多卖了52%。

仅仅因为它"存在"，就让杂志社多赚了3,432美元。这就是行为经济学中著名的"诱饵效应"。

原理很简单：人类不擅长判断一个东西的“绝对价值”，但极其擅长做"相对比较"。

当只有59美元和125美元两个选项时，你的大脑在比较"便宜 vs 贵"，大多数人选便宜的。

但当“125美元只买纸质版”这个诱饵一出现，你的大脑就不比较 A 和 C 了，它开始比较 B 和 C。

同样的价格，C 多了一个电子版。天哪，这不是白捡的吗！于是你心满意足地选了 C。

浑然不知自己刚刚多花了66美元——买了一本可能一辈子都不会翻开的纸质杂志。

这个套路如今无处不在。咖啡店的中杯定价，只是为了让你觉得大杯“更划算”。视频网站的月卡，贵到让你觉得年卡“不买就亏”。

手机发布会上，永远有一款“高价低配”机型，它唯一的使命，就是让旁边那款看起来“性价比极高”。

当你觉得自己占了便宜的时候，多半是有人精心摆放了一个诱饵，让你心甘情愿走进了更贵的那扇门。

那个没人选的选项，才是全场真正的主角。


1、[为什么沙子有粘性？](https://www.mentalfloss.com/posts/why-is-sand-sticky)

我们去海边玩，沙子会粘在皮肤、鞋子、衣服和头发上。

![](https://cdn.beekka.com/blogimg/asset/202304/bg2023042604.webp)

沙子的主要成分是二氧化硅，跟岩石一样，为什么会有粘性呢？

原来，沙子本身没有粘性，但具有亲水性，它会吸水。人体也是亲水的，在烈日下汗流浃背。当沙子接触到湿润的东西时，水分子之间就会产生粘性。

皮肤上往往还有油脂或者防晒霜，它们也会让沙子粘在皮肤上。

另外，皮肤还有一些微小褶皱，也会卡住沙子。

总之，想要去除沙子，就是等到皮肤变干，或者用水冲洗。

1、[我为什么离开谷歌](https://jayconrod.com/posts/122/leaving-google)

![](https://cdn.beekka.com/blogimg/asset/202211/bg2022112419.webp)

我加入谷歌时，被要求去负责谷歌文档的安卓 App。

这个产品原来由澳大利亚团队负责，我们从他们手里，拿到了 App 的控制权，但是没人跟我们交接，没人告诉我们应该如何处理这些代码、哪些地方需要注意等等。

我们团队都是新人，主要是应届毕业生，还有一些是从公司其他部门调来的。所以，大家都不了解这个 App 现有的技术基础设施。我们遇到问题，也没人可问。

谷歌的管理非常分散，我的经理同时监管大约25个项目。每过三周，我和她见面半个小时讨论工作。

就这样过了几个月，她都不记得我的项目名称。我看出来了，在现在的项目组里获得晋升是不可能的。

整个部门极度以产品为导向。产品经理做出所有决定，工程师对产品几乎没有影响，只是负责实现产品经理的决定。

很多功能需要两到三个季度来开发，但是排给我们的开发时间是两周。

这当然做不到，产品经理从那以后就要求我们，以30分钟为单位，报告自己每天在每项工作上的时间分配，让我感觉像机器一样。

所以，我就离开谷歌了。
