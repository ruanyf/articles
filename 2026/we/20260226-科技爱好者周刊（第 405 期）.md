1、[阿基米德级数](https://www.futilitycloset.com/2025/11/17/all-the-way-down-2/)

公元前200年，古希腊数学家阿基米德就发现，无穷级数 1/4 + 1/16 + 1/64 + 1/256 + … 的和是 1/3。

这可以从图中看出来。

![](https://cdn.beekka.com/blogimg/asset/202512/bg2025120407.webp)

上图是一个单位正方形，最大的黑色正方形面积为 1/4，第二大的黑色正方形面积为 1/16，以此类推。

黑色、白色和灰色区域在整个图形中所占面积相等，因此所有黑色正方形的总面积必定为 1/3。

还有另一张图。

![](https://cdn.beekka.com/blogimg/asset/202512/bg2025120408.webp)

上图中，如果最大三角形的面积为 1，那么最大的黑色三角形的面积为 1/4，第二大的黑色三角形的面积为 1/16，依此类推。黑色、白色和灰色区域在图形中所占比例相等，因此黑色区域的总面积必定为 1/3。

## 文摘

1、[为什么编号应该从零开始](https://www.cs.utexas.edu/~EWD/transcriptions/EWD08xx/EWD831.html)

作者：Edsger W. Dijkstra（著名计算机科学家）

有一个自然数序列，从2一直到12，我们可以写成 2, 3, ..., 12。

在程序中表达这个序列，我们不会使用三个点的省略号，而是用不等式，会有四种写法。

> - a） 2 ≤ i < 13
> - b） 1 < i ≤ 12
> - c） 2 ≤ i ≤12​​
> - d） 1 < i < 13

请问哪一种写法更好一些？

回答是 a 和 b 好于 c 和 d。理由是 a 和 b 的边界差异等于序列的长度。如果还有后续的序列，那么一个序列的上限等于另一个序列的下限。

a 和 b 里面，哪一个更好一点呢？

考虑到存在最小自然数，所以下限是可以封闭的，因此 a 是更好的写法，可以从最小自然数写起。

下一个问题是，当处理长度为 N 的序列时，我们希望通过下标区分其中的元素，起始元素应该分配什么下标值？

遵守 a 的写法，从下标 1 开始时，下标范围为 1 ≤  i  <  N +1；但是，从 0 开始，可以给出更精确的范围 0 ≤   i  <  N，上限正好等于长度。

因此，让序列编号从零开始，元素的序数（下标）等于它前面的元素数。
