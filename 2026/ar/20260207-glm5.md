# 智谱旗舰 GLM-5 实测：对比 Opus 4.6 和 GPT-5.3-Codex

## 一、引言

昨天，智谱突然发布了旗舰模型 GLM-5，完全出乎意料。

这么拼，非要赶在长假之前，上一个版本 GLM-4.7 发布还不到两个月啊……

![](https://cdn.beekka.com/blogimg/asset/202602/bg2026021001.webp)

GLM-4.x 在国内外评价已经很高了，公认是编程领域第一梯队的模型。这次新的大版本就让人很好奇，会有哪些改进。

巧的是，几天前，国外两个旗舰模型也发了新版本：Anthropic 公司发了 Claude Opus 4.6，OpenAI 公司发了 GPT-5.3-Codex。

这三个模型都主打编程，下面我就用实际的编程任务，比较一下它们的表现。

## 二、GLM-5 简介

官方的发布说明，这样介绍 GLM-5：**作为开源模型，GLM-5 完全对标顶尖闭源模型**，在两个地方做了特别强化。

**（1）复杂系统工程**

GLM-5 不单善于生成前端网页，更善于处理后端任务、系统重构、深度调试，摒弃了“重前端审美、轻底层逻辑”的模式。

> 它具备极强的自我反思与纠错机制，能在编译失败或运行报错时，自主分析日志、定位根因并迭代修复，直到系统跑通。

**（2）长程 Agent**

它能够跑长程任务，即多阶段、长步骤的复杂任务，可以自主拆分需求，自动化连续运行长达数小时，并保持上下文连贯与目标一致性。

**（3）小结**

GLM-5 可以完成的任务，已经超越了生成前端 UI，而是可以生成系统级大型复杂项目，比如操作系统内核、浏览器内核、V8 引擎之类的。

它的官方宣传语是“在大模型进入 Agent、大任务的时代，GLM-5 是你可以使用的开源选择。”

## 七、Laravel 转为 Next.js

最后一个测试是，将一个基于 PHP 语言 [Laravel 框架的 Web 应用](https://github.com/ruanyf/ai-test-case/tree/main/case04/starter)，转为 JavaScript 语言 Next.js 框架。

GLM-5 在处理时，几乎没有出现任何麻烦，很快就将 PHP 语言转成了 JS 语言，并且给出了转换后的代码结构。

![](https://cdn.beekka.com/blogimg/asset/202602/bg2026021035.webp)

它还在转化后，贴心地自动安装了依赖的软件包，做好了脚本编译，提示用户：你只要接入外部 API，一键执行`npm run dev`就能直接运行了。

![](https://cdn.beekka.com/blogimg/asset/202602/bg2026021036.webp)

我按照它的提示，运行很顺利，没有报错，打开`localhost:3000`就能访问应用了。

![](https://cdn.beekka.com/blogimg/asset/202602/bg2026021037.webp)

这是一个查看城市天气的应用。因为没有要求改变样式，所以看上去跟 PHP 原版一模一样。

右上角输入框，可以查询城市。

![](https://cdn.beekka.com/blogimg/asset/202602/bg2026021038.webp)

在候选城市中，选中你所要的城市。

![](https://cdn.beekka.com/blogimg/asset/202602/bg2026021039.webp)

点击进去，就是城市的详情页，有天气、日出日落时间、空气质量、地图等信息。

![](https://cdn.beekka.com/blogimg/asset/202602/bg2026021040.webp)

Opus 4.6 和 GPT-5.3 也生成了同样的结果，因为页面、功能完全一样，就不展示截图了。

值得一提的是，GLM-5 和 GPT-5.3 的转换时间都在5分钟左右，Opus 4.6 似乎遇到了一点问题，花费了整整20分钟。

这一轮单看结果，三个模型都很好，但是 GLM-5 花费的生成时间短，没有任何报错，全过程的用户体验好，我愿意投它一票。

## 八、总结

经过这些测试，GLM-5 的表现可圈可点，跟国外最新的旗舰模型在编程方面处于同一梯队，互有输赢。即使不如对手的地方，也是一些细节的差距。

这一次的新版本，除了优秀的网页设计，确实能感受到在特别强化的两个点“复杂系统”和“长程任务”有改进。

它生成的系统逻辑和后端代码，可靠性很不错，无论是生成过程，还是运行过程，报错都很少。最多就是一些功能的缺失，后期让 AI 再补上就可以了。另外，我有一项个人任务，它跑了足足两个小时，最后也完成了，没有乱掉。

我愿意把官方说的一段话，作为结尾。

> 2026年编程大模型正在从“能写代码”进阶为“能构建系统”，而 GLM-5 堪称开源界的“系统架构师”模型，从关注“前端审美”转向关注“Agentic深度/系统工程能力”，是 Opus 4.6 与 GPT-5.3 的国产开源平替。
