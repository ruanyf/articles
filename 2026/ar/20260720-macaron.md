# 如何基于 GLM 5.2 开发自己的模型，以 Macaron V1 为例

## 一、

大家关注衍生模型吗？

说实话，我以前是不关注的。基于其他模型加工出来的模型，能力都由底层模型决定，给人一种仿制品和正品的感觉，好像没啥可关注的。

但是，我最近遇到一个朋友，他让我的想法发生了变化。

他说，衍生模型反而更值得关注。因为，它不会削弱原始模型的长处，并且增加了功能，提升了表现，使用起来更顺手。

而且，对于模型开发来说，开发衍生模型其实是更合理的行为。别人花了大价钱训练出来的权重，免费开源出来，为什么不利用呢？就好像已经有现成的房子，你可以不花一分钱入住，何必一定要自己从头建呢。

听上去好像很有道理啊。

他进一步举例，他们现在就在做衍生模型，基于目前很红的 GLM 5.2 衍生出来了 [Macaron V1](https://macaron.im/mindlab)（马卡龙），做了微调和功能增强。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026072002.webp)

GLM 5.2 的评分已经很好了，他们修改以后，自己测了基准，分数还更高了。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026072001.webp)

上图中，蓝色是 Macaron V1，绿色是 GLM 5.2。

下面我就说说他们的做法，如何基于别人的模型进行开发。大家可以从中了解衍生模型是怎么来的。

## 二、

他们采用的方法叫做 Mixture-of-LoRA（MoL）。

根据我的理解，MoL 架构大概就是下图这个样子。这是我画的，不一定准确。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026072003.webp)

简单说，就是额外增加一个路由层，下面挂载了四个 1B 参数的 LoRA 专家（微模型）。

LoRA 是 Low Rank（低序）的缩写，表示这里的专家比 GLM 5.2 内置的专家，级别要低。

路由层收到用户输入的内容，会选择交给相应的专家，再将专家的处理结果与原始模型 GLM 5.2 的结果进行加权处理，得到最终结果。

这种结构的好处，就是专家是外挂的，可以根据需要进行增加，不会动摇已有知识。不同的专家还可以组合。缺哪方面的行业专家，直接外挂一个就行了，不需要改变原始模型。

有些同学可能已经看出来了，这种 MoL 架构跟“混合专家模型（MoE）”很像。确实如此，我觉得原理是一样的，不同之处是 MoE 是模型的内部专家，MoL 是模型的外挂专家。

## 三、

回到上面这张图，可以看到 Macaron V1 挂载了四个外部专家。

> - LoRA0：对话（chat）。
> - LoRA1：智能体（agent），可以理解成对应 OpenClaw 的能力。
> - LoRA2：编程（coding）。
> - LoRA3：可视化渲染（UI4A）。

这就是它为 GLM 5.2 优化的四个能力，目标是微调原始模型的行为，让它更符合用户需求。

这种外挂专家，不仅可以加强能力，还能改变原始模型的很多参数。

比如，GLM 5.2 的上下文窗口是100万 Token，但是 Macaron V1 加大到200万 Token。他们将这种方法开源了，项目叫做 [LongStraw](https://github.com/MindLab-Research/longstraw)，好像对于各种模型都适用，可以增大现有模型的上下文窗口。

读到这里，大家应该明白了，衍生模型不是原始模型的简单复刻，而是深度加工，可以改变行为和增强功能。

## 五、

接下来，测试代码能力。要求是生成一个 2D 的网页游戏《愤怒的小鸟》。

Macaron V1

GLM 5.2

首先，两者的 UI 很相似，小鸟和砖头的样式都很接近，说明 Macaron V1 确实是基于 GLM 5.2 衍生的。

但是，Macaron V1 明显实现得更好，不仅多出了砖头上面的小猪，而且撞击砖头的效果也更生动，游戏更强，说明外挂的专家还是起作用了。

## 六、

Macaron V1 还有一个 GLM 5.2 没有的特色功能，就是可视化渲染的专家 UI4A。它能够在 Web 窗口实时可视化展示模型的生成结果。

这个功能最好跟开源工具 [macaron-artifacts](https://github.com/mindverse-ltd/macaron-artifacts) 配套使用。它是一个定制的大模型 Web 界面，里面就实现了 UI4A 的可视化展示。

安装以后，它会起一个本地服务 `http://localhost:7878`，这就是模型自带的浏览器使用前端。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026072005.webp)

这个对话框的生成结果就是可视化展示，举例来说，你要求生成一个 3D 魔方，它就实时展示出来，用户还可以在窗口直接操作魔方。

作为对比，GLM 5.2 的[官方 Web 界面](https://bigmodel.cn/trialcenter/modeltrial/text?modelCode=glm-5.2)，同样的提示词只会展示代码和文字（下图）。

## 七、

经过上面的简单测试，可以看到，作为 GLM 5.2 的衍生模型，Macaron V1 不仅具有 GLM 5.2 的功能和表现，还做了深化和拓展，优化了代码的生成结果。

费用方面，两者的定价是一样的。因此，个人的日常使用可以考虑这个模型。它相当于 GLM 5.2 + 外挂的四个专家，而且这四个专家都针对个人生活场景（对话、Agent、代码和可视化渲染），实用性很强。

总之，**如果你要使用某个大模型，可以同时关注它的衍生模型，有时候可能是更好的选择**。

## Macaron-V1 模型简介

心洲科技旗下前沿实验室 Mind Lab 最新模型Macaron-V1

Macaron-V1 是 Mind Lab 面向个人智能体场景的开源模型，提供 Venti 与 Tall 两个版本。权重、官方 API 与配套工具均已上线。

V1 延续之前提出的 Mixture-of-LoRA(MoL)路线，基座升级至 GLM-5.2，原生支持 2M 超长上下文。

Macaron-V1 体现了我们对两种理念的押注：适应（adaptation） 与 协作（collaboration）。

我们相信，下一代 AI 的决定性因素，不再主要是扩展单一的单体模型，而是两种相辅相成的能力：其一，通过递归自我改进进行后训练——模型持续生成经验、从中学习，并改善自身行为；其二，多智能体协作——专业化模型与智能体构建在共享基础设施之上并相互组合。V1 是我们迈向这一未来的里程碑。

其中旗舰版本 Macaron-V1-Venti，在目前1T参数量级以下的模型中，提供了模型性能和推理成本的两者间的最理想选择
是继Macaron-V1-Preview之后推出的个人模型正式版。V1版本包含两个变体：

- Macaron-V1-Venti：748B（744B 基础模型 + 4×1B LoRA），我们的旗舰产品，也是全球首个基于 GLM-5.2 完成后训练的模型。

- Macaron-V1-Tall：35B，基于通义千问3.6后训练而来，专为本地部署打造。
