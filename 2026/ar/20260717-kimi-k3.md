# 怎么看待“史上最大开源模型” Kimi K3？

## 一、

这两天，大家都被 [Kimi K3](https://www.kimi.com/blog/kimi-k3) 刷屏了吧，到底应该怎么看待这个史上最大开源模型？

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071702.webp)

我来说说我的看法，分成几点。

**首先，它的性能是否真的接近 Fable 5？**

根据多个国外机构的测试，以及我自己的测试（附在后文），我认为是真的。

Arena AI 把它列为互联网开发的[第一名](https://arena.ai/leaderboard/code/webdev)。注意，这只是比较前端编程，其他方面还是 Fable 5 领先。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071706.webp)

独立评测机构 Artificial Analysis 把它列为[第三](https://x.com/ArtificialAnlys/status/2077832874183860404)。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071804.webp)

其他的几种测试，比如办公能力 [GDPval v2](https://artificialanalysis.ai/#gdpval)、写作能力 [Writing Elo](https://x.com/Whats_AI/status/2077860441380798908)、氛围编程 [BridgeBench](https://x.com/bridgebench/status/2078109314074443846)、全栈编程 [Next.js](https://nextjs.org/evals) 的测试，它都排名前三甲，甚至第一。

我认为，可以确定它自己在发布公告中的定位是可信的，即实力略逊于 Claude Fable 5 和 GPT 5.6 Sol，但是强于 Claude Sonnet 4.8。

## 二、

其次，**为什么它的能力突然变强了？**

它的上一个版本 Kimi K2.7 Code 是6月12日发布的，表现只能说是普通，在 Artificial Analysis 排名中仅仅位列17。

为什么仅仅过去了一个月，就有这样巨幅的提升呢？

比起以前的版本，K3 最大不同就是参数数量从 1T 增加到了 2.8T，这大概就是它能力飞跃的原因。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071701.webp)

这个 2.8T 的参数数量是空前的，远远多于其他开源模型，足以跟国外旗舰比肩。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071703.webp)

我们知道，参数数量代表了大模型计算 Token 时，所要考虑的因素。参数越多，就表明模型所考虑的因素越多，一般来说，生成的效果就越好。

## 三、

那么，**它怎么做到一下子增加这么多参数？**

我们知道，参数越多，意味着计算量越大，从而模型的使用成本会大幅上升，响应时间也会变长。

所以，真正困难的不是增加参数，那只要为方程式拟合更多变量就行了，而是如何控制计算量，不让它随着参数数量增长而爆炸。

根据他们的发布报告，我是这样理解的，Kimi K3 为了减少计算量，充分利用了“专家混合模型”（MoE）架构，也就是每次计算 Token 的时候，不使用全部参数，只激活一部分参数，这样就避免了大部分的计算。

这一次，Kimi K3 没有公布激活参数的数量，发布报告只是提到这样一段话：

**性能**

提高代币效率并提升智能水平：Kimi K3 在人工智能分析智能指数上的代币使用量显著降低，比 K2.6 减少了 21% 的输出代币。新模型完成全部九项评估仅使用了约 1.32 亿个输出代币，而 K2.6 则使用了约 1.66 亿个，同时还取得了更高的分数。

**价格**

第一方 API 的定价为每 100 万输入/输出令牌 3.00 美元/15.00 美元，

每百万个输入代币3美元，每百万个输出代币15美元，与Anthropic的Claude Sonnet系列处于同一价格水平，使其成为迄今为止中国人工智能实验室发布的最昂贵模型。


百万 Token 输入输出价格 20元/100元。比上一个版本 K2.7 Code 的 6.7元/27元 翻了好几倍。

GLM 5.2 的 8元/28元 贵了许多。

Qwen 3.7 Max 的 12元/36元。

DeepSeek V4 Pro 3元/6元

每次任务成本（0.94 美元）与 GPT-5.6 Sol（1.04 美元）相近，约为 Opus 4.8（1.80 美元）的一半，但高于其他开放权重模型。

Moonshot AI 为 K3 定价远高于其 K2 定价（K3 的输出代币价格为 15 美元/百万代币，而 K2.6 为 4 美元）。

因此，就每次任务成本而言，该模型比 Opus 4.8 更便宜，与 GPT-5.6 Sol（1.04 美元）相近，但高于其他开放权重模型，例如 GLM-5.2（0.32 美元）和 DeepSeek V4 Pro（0.04 美元）。




# 史上最大开源模型 Kimi K3，能对抗 Fable 5 吗？

## 一、

Kimi 突然发布了[新的旗舰模型 K3](https://www.kimi.com/blog/kimi-k3)。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071702.webp)

这可真意外，因为上个版本（K2.7 Code）的发布仅仅过了一个月（6月12日）。但是，相比迭代速度，更意外的是 K3 的架构发生了巨变。

大家看看下表，就明白这个变化有多巨大了。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071701.webp)

可以看到，K3 的参数总数达到了 2.8T，是上一代的近三倍！

这可是一个闻所未闻的数字，我从未听说哪个模型有这么多参数。国外的旗舰模型，比如 Claude 和 GPT，从不公布参数数量，但一般猜测，它们的参数大概就是一点几个 T，那也比 Kimi K3 的 2.8T 小。

至于开源模型，就更没有这种级别的怪兽了。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071703.webp)
 
参考上表，Kimi K3 是史上最大的开源模型，规模远远超过其他模型。

需要说明的是，它还没放出源码，根据[发布说明](https://www.kimi.com/blog/kimi-k3)，权重文件会在7月27日开源。

## 二、

不熟悉大模型的读者可能会问，参数有什么意义？参数数量多，又说明了什么？

简单说，参数就是大模型计算 Token 时，所要考虑的因素。参数越多，就表明模型所考虑的因素越多，一般来说，生成的效果就越好。

Kimi K3 的参数这么多，表明生成效果肯定比上一代要好得多，也应该比那些小规模的模型好。

著名的国外大模型评测网站 Arena AI，已经把 Kimi K3 评为互联网开发的[第一名](https://arena.ai/leaderboard/code/webdev)，超过了 Claude Fable 5、GPT 5.6 sol 等国外旗舰模型。

![](https://cdn.beekka.com/blogimg/asset/202607/bg2026071706.webp)

那么，参数是否越多越好呢？

也不能这么说，参数多是有代价的，就是计算量会增多，导致成本上升，响应速度下降。

好在 Kimi K3 采用的是混合专家模型（MoE）架构，计算 Token 时不会读入所有参数，只会激活一部分参数，这就大大减少了计算量。

目前，Kimi K3 没有公布激活参数的数量，但是发布公告有这样一段话：

> “我们还扩展了专家混合模型 (MoE) 的稀疏性，当与 Stable LatentMoE 框架结合使用时，能够有效激活 896 位专家中的 16 位。结合改进的训练和数据处理方法，这些结构性变化使得整体扩展效率相比 Kimi K2 提升了约 2.5 倍，从而使模型能够更有效地将计算转化为智能。”

根据这段话，Kimi K3 有896位专家，相当于每位专家对应 3.13B 参数（2800/896），那么激活16位专家，就相当于激活 50B 参数。

这段话还说，k3 比 k2 的效率提升2.5倍。K2 共有 1T 参数，激活 32B，效率是 3.2%，除以2.5就是1.28%，相当于 2.8T 参数激活 36B。

不管 50B 还是 36B，都表明 Kimi K3 是一个相当稀疏的模型，实际计算量并不像表面看上去那么大，所以既能增大参数，又能保持成本的可控性。
