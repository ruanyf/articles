# 当办公套件遇上 AI Agent：开源项目飞书 CLI 介绍

## 一、引言 

今年春节前，龙虾 OpenClaw 突然爆火，引发各种云服务纷纷开放 API 让它接入。

有一类非常重要的软件，也在这期间开放了 API，但是没有得到外界的太多关注。那就是办公套件。

![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051409.webp)

大家想象一下，办公文档一旦实现 AI 自动化，可以简化多少工作和流程，而且是多大的一个市场！

我最近就有这方面的需求，想让 AI 在云端自动化管理文档，就去搜索了可以接入 Agent 的办公套件工具包。

下面就是我的搜索结果。

- [Google WorkSpace CLI](https://github.com/googleworkspace/cli)
- [飞书 CLI](https://github.com/larksuite/cli)

- [企业微信 CLI](https://github.com/WecomTeam/wecom-cli)

这四个开源工具包都允许用户在命令行调用 API，操作自家的云端 SaaS 服务。

其中，谷歌 WorkSpace CLI 是半官方的，是他们的一个工程师个人在维护，其他三个都是官方项目。

这三个国产办公套件工具包之中，[飞书 CLI](https://github.com/larksuite/cli) 比较突出。

我用了几天[飞书 CLI](https://github.com/larksuite/cli)，感觉不错，如果你也想**办公套件接入 Agent、实现办公自动化**，它可能是这三家里面最容易满足你的需求的工具。下面就是我的简单使用介绍。

## 二、基本情况

首先，飞书 CLI 是一个很新的工具，今年3月28日刚刚发布，迄今不过40多天。

它的活跃程度非常高，已经发了30个版本，基本上（工作日）一天一个新版本。如果在仓库提问或请求功能，很快会得到响应，这一点比较让人放心。

![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051401.webp)

其次，这个工具包的功能相当全，飞书现有的业务域，它几乎都能操作，已经具备100多项能力。

![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051402.webp)

上面是它现在支持的17个业务域，囊括了企业办公的各方面。我认为，它的最大卖点就是这个：可以开箱即用某一个业务域、某一项能力的 API，也可以深度编排多种能力，**定制跨域工作流**，实现复杂业务的自动化。

最后，它是一个彻底开源的国际化项目，采用 MIT 许可证，几乎没有使用限制。外部的代码贡献者数量不少，我看有不少外部 PR 被合并进了主干。

## 三、项目结构和 Skill 封装

在代码上，这个项目采用多层封装的结构。

最底层是[飞书 OpenAPI](https://www.feishu.cn/content/785259988660)，可以看到每一个具体的 API 接口，包括参数、返回值、报错类型等等。

第二层是命令层，每一个命令与 OpenAPI 的定义一一映射。

第三层是快捷方式层，对命令层进行了包装，提供默认参数，简化调用写法，输出便于阅读的格式，尽量对用户友好，降低使用门槛。

除了这三层，还有一个单独的 SKill 部分，也就是为 Agent 封装的能力扩展包，包括专门的脚本和提示词，大大方便了 AI 的调用，目前内置了24个 Skill，都属于官方的最佳实践。

![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051403.webp)

## 四、AI agent 的接入

这个项目使用 Go 语言开发，封装为 npm 软件包，[安装](https://github.com/larksuite/cli#quick-start-human-users)采用 npm 命令。

```bash
$ npx @larksuite/cli@latest install
```

不过，更好、更简单的安装方法，是[使用自然语言](https://www.feishu.cn/feishu-cli)让 Agent 自己去安装。


![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051404.webp)

整个安装过程是自动的，但是登录和授权需要手动完成。

![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051406.webp)

安装后，Agent 就能控制你的飞书账户了，这时就可以直接下达任务让 AI 去跑。

## 五、典型能力

这个工具能做的事情很多，下面罗列几项。我看官方有一个[《飞书 CLI 的 100 多个新本领》](https://bytedance.larkoffice.com/wiki/CLNjwBozvi11IjkeChOcltHinye)的能力清单，大家可以参考。

（1）文档功能。我平时写的是 Markdown 文档，但飞书是可视化编辑器，你可以把 Markdown 原稿提交给 AI，让它调用 Skill 进行排版和发布。

> [提示词]
> 把这篇 Markdown 内容创建成飞书文档，排版要好看。

飞书支持文档评论，我们一般用这个功能发表修改意见，可以让 AI 根据评论自动修改文档。

> [提示词]
> {{文档链接}} 根据评论修改文档，修改后，用划词评论标识出修改点。

也可以让 AI 生成文档的修改意见。

> [提示词]
> {{文档链接}}阅读一下这篇文档，看下作为提供给外部用户的说明文档，是否足够清晰、简洁。不要直接改文档，只把你觉得有优化空间的位置划词评论出来，在评论里写上修改意见。

（2）画板功能。画板主要用于生成可编辑的流程图、框架图、时序图等，可以让 AI 根据文档内容，生成相应图形插入文档。

![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051407.webp)

（3）幻灯片功能。它提供预置的飞书 PPT 模板，目前有40多个，足够用了，你可以指定模板，将文档转成幻灯片。

![](https://cdn.beekka.com/blogimg/asset/202605/bg2026051408.webp)

（4）工作流。工作流就是一个跨多个业务域的多步骤操作流程，比如让 AI 获取某个群聊的参与者名单 --> 参看每个参与者的日历，找到所有人空闲时间 --> 为这些人约一个在线会议。

> [提示词]
> 帮我看一下【XX】群里所有人的日历，然后下周找一个大家都合适的时间开一小时的讨论会

再比如，让 AI 读取邮箱，一旦有重要的未读邮件，就转发到讨论群。

> [提示词]
> 查看我所有的未读邮件，重要邮件发摘要到【XX】项目群

如果你已经是飞书用户，我认为这个[飞书 CLI](https://github.com/larksuite/cli) 是必备工具，搭配 AI agent，会大大提高办公套件的实用性和便利性。

如果还没用飞书，考虑到它是免费的，所有基础设施都已经通过这个工具包向 agent 开放了，不用似乎有点可惜。

看得出来，官方非常重视这个项目，投入很大，一直在密集开发，想要抢占 Agent 市场。我觉得，他们应该会长期做下去。

再加上，项目本身是 MIT 许可证，可以商用，所以也是一个机会，值得大家利用飞书的基础设施，开发二级应用，探索一下企业级办公软件市场。

## 历史

3 月 28 日，飞书正式开源 「飞书 CLI」（larksuite/cli）。该项目由 Go 语言开发，通过 npm 分发，GitHub 仓库为 larksuite/cli，采用 MIT 许可证，发布首日即获得 1000+ Star。

目前，飞书 CLI 已经是当前国内工作平台官方 CLI 中最受关注的项目。9,614 stars、648 forks、218 个 open issues、28 个 releases（数据快速增长中，以发布前为准），这组数据同时说明了四件事：关注度高、转发意愿强、真实使用密度高、项目迭代快。

## 飞书CLI的技术优势

- 架构上，飞书 CLI 设计了「三层命令架构」，真正做到了Agent友好：

第一层 Shortcut 快捷命令：面向人和 AI 共同优化，提供智能默认参数、友好输出和 dry-run（试运行） 预览，降低调用门槛。

第二层 API Commands：与飞书 OpenAPI 一一映射，同时具备动态发现能力。

第三层 Raw API Calls —— 即便在任何老版本 CLI 中依然可以访问所有飞书最新开放的 OpenAPI 能力，理论上可以做到无限扩展。这一层也可以让模型充分发挥智能：当 Agent 遇到复杂任务或预设命令覆盖不到的场景时，可以直接通过对 Raw API 的理解，自己尝试找到各种可能的解决方案。

- 在命令架构之上，飞书CLI还提供了 Skills —— 给 Agent 预制的能力扩展包。每个 Skill 封装了使用飞书 CLI 完成某类典型场景任务的最佳路径。这解决的是一个很现实的问题：开放 API 的数量多，并不等于 AI 用得好。Agent 面对几百个命令，往往不知道怎么组合才最顺。Skills 把「怎么正确使用」这些 API 的方法沉淀成 Skill。Agent 不用在每次任务里重新摸索最佳实践，而是直接站在飞书已有的积累上，让 AI 从「能用」跨到「好用」。

社区每天真实的 Issue 和 PR，让产品每天都在被真实场景打磨。开源迭代的好处是收敛到正确的速度，以及开源开发者们的开发速度加成，这在 AI 时代是压倒性的优势。
