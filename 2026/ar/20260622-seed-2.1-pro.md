# 豆包背后的 Seed 模型，又升级了

## 一、引言
 
使用[豆包](https://www.doubao.com)的时候，你注意过它背后的模型吗？

我想很少人注意吧，整个网站只在“设置”部分提到一句“关于豆包大模型”。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062301.webp)

点击就会跳转到字节的云服务平台“[火山引擎](https://www.volcengine.com/product/doubao)”，那里对外提供豆包大模型的 API。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062401.webp)

这个模型是字节自研的，正式名字是 Doubao-Seed-xxxx。

Seed 是字节跳动的人工智能团队，前一段日子很火爆的视频模型 Seedance 就出自这个团队。豆包大模型（Seed 基座模型）是他们的旗舰模型。

我一直很好奇，跟其他模型相比，Seed 模型到底是什么水平，有怎样的能力，毕竟它决定了豆包的上限。

## 二、Seed 2.1 pro

昨天，我看到朋友圈，火山引擎在它的年度大会发布了最新版 Seed 2.1 系列模型。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062302.webp)

这个系列一共两个模型，最强的是 Seed 2.1 Pro，当天就在 TRAE 和扣子全量上线，并且 API 也通过[火山方舟](https://console.volcengine.com/ark/region:ark+cn-beijing/overview?briefPage=0&briefType=introduce&type=new)开放，对外提供服务了。就在发稿前，豆包[最新公告](https://mp.weixin.qq.com/s/Sb-NMXTrWFQES1EDO_Gr2g)推出专业版，在办公任务里已经能全量使用 Seed 2.1 模型了。

[发布报告](https://mp.weixin.qq.com/s/a-eevQvWVAYwZQeFi1ZomA)说的很清楚，这个模型跟其他模型有所不同。一般来说，模型都有自己的强项，有的擅长编程，有的擅长推理分析，有的擅长作图。

但是，Seed 2.1 Pro 不一样，**它是一个均衡的模型**。所谓“均衡”，就是各种事情都能做，而且每件事做得都还可以。这其实很合理，它用来支撑豆包，而豆包的定位就是一个普遍适用的日常 AI 应用，肯定要求背后的模型能力均衡。

发布报告明确说，这个版本提升了代码能力（coding）和代理能力（agent），同时保持领先的视觉能力（VLM）。

对于普通用户来说，一个模型如果同时达到这三个能力，那么日常90%以上的需求就都能满足，不必切换模型了。平时可以都用它，如果遇到专业性强的困难任务，再考虑临时切换专业模型。

我当天第一时间就接入 API，测了它的这三种能力。

## 三、代码能力

我先测了 Seed 2.1 Pro 的代码能力。

就在我测的时候，国外模型评分网站 arena.ai 的排名已经出来了，在 [WebDev 榜单](https://arena.ai/leaderboard/code/webdev)上，Seed 2.1 Pro 排名世界第八。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062312.webp)

这个排名意味着，Seed 2.1 Pro 的互联网开发能力非常强。要知道进入前十名的国产模型只有两个，其他都是国外的旗舰模型。

根据这个排名，它的代码能力与 Claude Opus 4.6（2026年2月发布）相当，略逊于 Opus 4.7（2026年4月发布）。

我还是用我的那几道[网站编程题](https://github.com/ruanyf/ai-test-case)，测试它编写网页的能力，这样便于比较。

（1）网页重构测试，重构一个简陋的网页。它的生成结果如下。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062313.webp)

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062314.webp)

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062315.webp)

生成的模拟器整体效果不错，界面简洁易用，动画流畅，缺点是天体都是 2D 的，不是 3D，虽然不影响使用，但就不够逼真。

从这几个测试来看，Seed 2.1 Pro 的前端 UI 能力是可以的，完全拿得出手。至于其他代码能力，尤其是后端编程，由于时间关系没测，我只能说如果能达到国外评测的那样，与 Opus 4.6 相当，那就足以日常使用了。

一般认为，Claude Sonnet 3.7 和 Claude Opus 4.6 是大模型从实验室到生产环境的质变点。现在，Seed 2.1 Pro 也达到了这个点，开始迈向生产级的代码生成了。

## 四、代理能力

代理能力（agent）指的是，模型代表用户，操作计算机的能力。

OpenClaw/Hermes 这样的 AI 代理工具，就最好搭配 Agent 能力强的模型，让其在底层进行分析和思考，发出指令，再由 Agent 软件执行。

六月初，豆包推出一个“任务模式”，现在想起来，这就是为 Seed 2.1 Pro 做准备。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062304.webp)

这个任务模式其实就是 Agent 模式，让豆包化身为 OpenClaw（或 Hermes）去操作计算机，完成具体的任务。

它分为网页版和本地版，底层都是 Seed 2.1 Pro，区别仅仅在于本地版是直接操作你的个人计算机，网页版则是在服务器操作虚拟机里的系统。

我这次主要测模型，就不测豆包的本地 Agent 功能了，只看一下网页版的办公任务处理。

（5）Word 文件测试。

在豆包官网，点击左侧的新建办公任务，输入任务描述：“将微博热搜的截图放进一个 Word 文件”。

豆包就会在云端虚拟机里打开微博，然后创建截图，生成 Word 文件。页面实时显示每一步的进展。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062305.webp)

Word 文件生成完毕后，页面就会提供下载。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062306.webp)

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062307.webp)

整个任务执行过程非常清晰，很容易就得到了结果。

我上传某公司的季度报告（PDF 文件）和财务报表（Excel 文件），要求豆包提取关键信息，生成 PPT 文件，测试模型分析资料和生成报告的能力。

## 五、视觉能力

视觉能力指的是，模型可以理解图像和视频。只有多模态模型才具备视觉能力，单纯的文字模型没有这项能力。

据说，Seed 模型的视觉能力一直是领先的。我正好前些天测过其他模型的视觉能力，就直接复用了上次的题目。我让 Claude Code 接入 Seed 2.1 Pro 的 API，然后把视频或图像拖入提示框就可以了。

（7）图片理解能力。请问下面的图片中，有几块红色积木？

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026060311.webp)

Seed 2.1 Pro 的回答如下：

> 图片里一共有5块红色积木，分别是：
> 
> 1. 木板左侧外面的红色五角星积木
> 2. 木板上方放置的红色正方体积木
> 3. 插入木板孔中的红色五角星积木
> 4. 插入木板孔中的红色正方体积木
> 5. 木板右侧外面、绿色圆柱前方的红色半圆/椭圆柱形积木

不错，它不仅给出了数量，还给出了积木的准确形状和位置。

（8）视频理解测试。请问[这段视频](https://www3.cde.ca.gov/download/rod/videosample1.html) 0:16 处的小动物是什么？

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026060313.webp)

Seed 2.1 Pro 的回答如下：

> 视频 0:16 处出现的是开源 3D 动画《大雄兔（Big Buck Bunny）》中的配角 Flaps，是一只圆滚滚的卡通紫灰色小鸟：它有着蓬松的紫灰色绒毛、橙黄色的鸟喙、大眼睛，还有橙色的小爪，是片中三只爱恶作剧的小动物角色之一。

它不仅总结了画面，还认出这是哪一部动画片的角色。

（9）视频页面复制。在网页复现这段视频的[水墨效果](https://x.com/hayashimon1/status/2064658158698782873)，呈现出墨水在紙上自然晕染、流动和融合。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061408.webp)

这道题的难度比较高，原始动画是用 Fable 5 模型生成的，动画效果是随机的。下面就是 Seed 2.1 Pro 生成结果的截图，它的生成速度倒是很快。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026062303.webp)

可以看到，它复刻了 UI，也做出了墨滴的流动效果，但缺少原始视频自然飘逸的感觉。这应该跟它逐帧复制原始视频，没有直接用随机的水滴流动算法有关。不过，单纯从视频理解的角度看，它确实是按照原始视频在做。

从这些测试来看，Seed 2.1 Pro 的视觉能力相当好，理解图像和视频完全不成问题。

## 六、总结

以上就是 Seed 2.1 Pro 的三大能力——视觉、代理、代码——的简单测试。

它的表现确实均衡，在任何一个方面都还可以。我认为，一个值得关注的亮点是 Agent 能力。考虑到豆包的海量用户，它配合豆包的任务模式，市场其实比 OpenClaw / Hermes 更大，带给开发者更多的机会。

接下来，我还会在日常编程中继续测试它。大家也可以试试它，火山引擎的文档有各种第三方工具[接入它的 API 方法](https://www.volcengine.com/docs/82379/2160841?lang=zh)。

“Seed 大模型”通常指的是字节跳动 Seed 团队（ByteDance Seed）所研发的一系列前沿大模型。

Seed团队是字节跳动将重点放在探索通用人工智能（AGI）和下一代基础模型的核心研究团队。他们的工作主题是从多模态理解、视觉与视频生成、到最新的大语言模型推理等多个全渠道前沿方向。

核心大语言模型：豆包大模型2.0（Doubao-Seed-2.0）
字节跳动将种子团队在特工与强化学习领域的最尖端研究封装到了新一代大模型中。

- 多模态视觉理解：Seed-VL系列（如Seed1.5-VL）
- 视觉与3D生成：Seedance、Seedream 与 Seed3D

6月23日，火山引擎将在年度Force大会上，发布最新基模Seed-2.1系列（Pro、Turbo、Evolving），API同步上线火山方舟。当天，豆包专业版-办公模式、TRAE、TRAE WORK、扣子均会上线Seed-2.1-Pro。

相比今年2月发布的Seed-2.0，Seed-2.1系列整体在Coding和Agent能力上显著提升，VLM 能力（视觉）持续领先。

截止6月17日，基于Seed-2.1-Pro过程版API的众测和评测集显示，Seed-2.1-Pro多数任务上超过Opus 4.6，相比Opus4.7有差距。By场景来看，Seed2.1在General Agent/生产力/Office任务上全面超过Opus 4.6，在Coding任务上强在执行。

过去一年，全球范围内模型的迭代不是对着静态的benchmark优化、也不是按照版本号线性优化，而是存在质变点。跨越这个质变点，模型会快速在很多场景中变得可用、好用。这是过去一年MaaS（模型API）生意迎来ARR暴涨的根本原因。比如，在视频生成领域，Seedance2.0是一个质变点；在基模这条线上，Claude Sonnet 3.7和Claude Opus4.6分别是可以做Agent、和Coding端到端长程任务的质变点。

现在，Seed-2.1正在跨越质变点，迈向生产级可用。

## 评测入口

【Coding任务】：Claude Code、Codex等
【生产力任务】：豆包专业版-办公模型、OpenClaw等

豆包的办公任务模式
