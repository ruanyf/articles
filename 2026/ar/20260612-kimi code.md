## 五、/goal 命令

Kimi Code 最近的更新，支持了`/goal`命令。我就来测这个功能，看看这种高级命令是不是好用。

所谓`/goal`就是为大模型设置一个具体的目标，一旦实现该目标，模型就立即停止运行。

之所以要引入这个命令，是因为提示词很多时候过于模糊，只是一个笼统的描述，导致大模型反复思考，发起多轮循环，不停地要求用户确认，非常费事。

有了`/goal`命令，大模型就有了明确目标，理论上可以自主运行，不需要用户介入，一旦预设的条件满足就停止。

下面是一个简单测试，我找了一张体积很大的图片，足足有 2.1MB。然后，输入下面的提示词。

> /goal 重新调整这张图片的尺寸，使其体积刚好小于 120KB。

注意，最前面是`/goal`命令，而这段提示词本身是一个非常具体的目标。

这时，Kimi Code 会让你选择，是否允许它完全自主运行，还是某些操作依然要求你来批准。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061401.webp)

选择第一个选项“Switch to Auto and start”（完全自主运行），这时没有任何需要你批准的操作，你可以走开干其他事情。

下面就是 Kimi Code 的运行结果。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061402.webp)

> ● 已完成图片尺寸调整。
>
>   • 原图：6629 × 5504 像素，约 2.1 MB
>   • 调整后：948 × 787 像素，122457 字节（约 119.59 KB）
>   • 文件路径：/Users/ruanyf/del-202606/demo.jpg
>
>   最终文件体积刚好小于 120KB，并通过 python3 读取文件大小进行了验证。

可以看到，它把图片体积调整到 119.59KB，刚好满足“小于 120KB”的要求。

整个过程完全不需要人工介入，非常丝滑。如果你需要让大模型长时间运行某个任务，非常推荐使用`/goal`命令。

## 六、Kimi Datasource

Kimi Code 的大部分功能，都跟 Claude Code 相同，但也有一些独家功能。

其中，我最喜欢的就是 [Kimi Datasource 插件](https://www.kimi.com/code/docs/en/kimi-code-cli/customization/plugins.html#kimi-datasource)。

> Kimi Datasource 是 Kimi Code 的官方数据插件。它支持以自然语言查询金融市场数据、宏观经济指标、公司注册记录、学术文献以及中国法律法规——无需手动调用 API 或注册数据账户。（摘自官网）

它是一个数据源插件，连接了同花顺、天眼查等数据库，可以免费查询实时股票价格、财报、学术论文等，非常实用。

安装很简单，就是输入`/plugins`，选中 Marketplace 进入 Kimi 的插件市场，选择 kimi-datasource。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061403.webp)

然后，输入`/reload`激活插件，就可以了。

使用时输入`/kimi-datasource [你的问题]`，以查询论文、财报和公司信息。

> /kimi-datasource 减肥药副作用的最新研究

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061404.webp)

> /kimi-datasource 贵州茅台的最新利润表

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061405.webp)

> /kimi-datasource 字节跳动有哪些股东

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061406.webp)

总之，这个插件可以为大模型引入最新数据，支持多种数据源，推荐使用。

## 七、总结

经过这些测试，我对 Kimi Code 的印象很不错，非常好用的编程代理，确实可以替代 Claude Code。

至于它搭配的 Kimi K2.7 code 模型，作为一个万亿参数级的模型，使用效果不错，运行非常流畅，顺利完成了各种任务。

给我印象最深的是，它的 Token 消耗相对节省，上面这么多测试跑下来，只消耗了会员计划的一小部分。

[官方介绍](https://www.kimi.com/resources/kimi-k2-7-code)这个版本的推理能力进行了“瘦身”，Token 消耗平均比上个版本减少30%。它的优势本来就是性价比高，这次又强化了这个优势，值得看好。

（完）

# Kimi Code 可以当做 Claude Code 的平替吗？

## 一、引言

自从 Claude Code 一飞冲天，编程代理似乎成了所有大模型公司的标配，一时之间百花齐放。

虽然我是 Claude Code 的用户，但随着越来越多使用国产大模型，也不禁跃跃欲试，想尝试一下国产编程代理。

我这一次选择的目标，就是月之暗面公司的编程代理 [Kimi Code](https://www.kimi.com/code)。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061204.webp)

它完全对标 Claude Code，我看到的国内外评价都很不错，有人甚至说，它就是后者的平替。

下面就是我的试用体会，我会比较两者的相似度，以及用它们解决同样的编程问题。

## 二、Kimi Code

Kimi Code 最早是 Kimi 模型的命令行终端，代码[开源](https://github.com/MoonshotAI/kimi-cli)，在 GitHub 上的星星有 9K。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061205.webp)

今天1月，它升级成了独立产品 Kimi Code。这两周开始加速开发，[发布日志](https://www.kimi.com/code/docs/kimi-code/whats-new.html)几乎每天更新。

它搭配了自家的旗舰模型 K2.6，据说具有极强的编程能力。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061206.webp)

这一点跟 Claude Code，后者也是强调自己极致的编程能力。

下面，我就开始比较。需要说明的是，我这次比较的是编程代码，不是模型本身，所以 Claude Code 的后台模型也采用 Kimi K2.6。

## 三、相似性
