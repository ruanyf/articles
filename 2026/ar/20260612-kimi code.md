# Kimi Code + K2.7 代码能力测评

## 一、引言

Kimi 最新的旗舰模型 K2.7（准确说是 K2.7 Code 分支） 上周发布了，拥有这个系列最强的代码能力。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061301.webp)

同时更新的，还有配套的编程代理 [Kimi Code](https://www.kimi.com/code)，号称对标 Claude Code，正好一起测。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061204.webp)

Kimi 模型一向低调，但是业内的口碑非常好，很多人推荐，能力强，价格低，性价比突出。

我第一时间就上手测试了，看看 Kimi Code + K2.7 这个组合，到底表现怎样，能不能替代 Claude Code。

## 二、Kimi Code

如果你以前用过 Claude Code，那么 Kimi Code 上手毫无问题，两者相似度极高。

安装方式和用户界面，非常类似。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061302.webp)

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061104.webp)

上面第一张图是 Kimi Code，第二张是 Claude Code，是不是看上去很像？

事实上，它们的命令也是兼容的。Kimi Code 支持 Claude Code 几乎所有的操作命令，反正我还没遇到不支持的情况。

甚至运行过程中“权限批准”的提示都很相似。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061303.webp)

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061306.webp)

上面第一张图是 Kimi Code，第二张是 Claude Code。

总之，如果不是事先知道，光看 UI 可能会分不清这两个编程代理。

# Kimi Code 可以当做 Claude Code 的平替吗？

## 一、引言

自从 Claude Code 一飞冲天，编程代理似乎成了所有大模型公司的标配，一时之间百花齐放。

虽然我是 Claude Code 的用户，但随着越来越多使用国产大模型，也不禁跃跃欲试，想尝试一下国产编程代理。

我这一次选择的目标，就是月之暗面公司的编程代理 [Kimi Code](https://www.kimi.com/code)。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061204.webp)

它完全对标 Claude Code，我看到的国内外评价都很不错，有人甚至说，它就是后者的平替。

下面就是我的试用体会，我会比较两者的相似度，以及用它们解决同样的编程问题。

## 二、Kimi Code

Kimi Code 最早是 Kimi 模型的命令行终端，代码[开源](https://github.com/MoonshotAI/kimi-cli)，在 GitHub 上的星星有 9K。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061205.webp)

今天1月，它升级成了独立产品 Kimi Code。这两周开始加速开发，[发布日志](https://www.kimi.com/code/docs/kimi-code/whats-new.html)几乎每天更新。

它搭配了自家的旗舰模型 K2.6，据说具有极强的编程能力。

![](https://cdn.beekka.com/blogimg/asset/202606/bg2026061206.webp)

这一点跟 Claude Code，后者也是强调自己极致的编程能力。

下面，我就开始比较。需要说明的是，我这次比较的是编程代码，不是模型本身，所以 Claude Code 的后台模型也采用 Kimi K2.6。
